Purpose

here in NEWGEO() we re-compute some grid metrics, more precisely:

a) measures of the triangle/tetrahedra (LVOL) are stored in VOL(NELEM+1:2*NELEM)

b) measures of the median dual control volumes (LMEDIAN) are stored in MEDIAN(NP+1:2*NP)

c) face normals (LFACNOR) are overwritten on the current ones

d) cell to face pointers (LCELFAC) are overwritten on the current ones, but should not change anyway, since it is assumed that the connectivity is un-changed

e) grid velocities LXYZDOT

this has to be done BEFORE calculating the residual i.e. before calling RHSFunction

in NEWGEO all metrics are updated at some time which is intermediate btw n and n+1 the areas/volumes of the median dual CVs are however those at n+1 since these are only needed to update the solution, not to compute the residual

Parameters
[in]ICELNODCell to node pointer: ICELNOD(i,j) gives the global node number of the i-th vertex of the j-th element
[in]ICELCELCell to cell pointer: ICELCEL(i,j) gives the element number that shares the face opposite the i-th vertex of the j-th element if 0 or > NELEM, that face is a boundary face
[in]PMAPMap for periodic nodes
[in]ICELFACCell to face pointer: ICELFAC(i,j} gives the global face number of the face opposite the i-th vertex of the j-th element
[in,out]FACNORCartesian components of the normals to a face, multiplied by the face area
[in,out]VOLarea/volume of the simplicial elements (triangles,tetrahedra)
[in,out]VMEDIANarea/volume of the median dual cells
[in,out]CORGCartesian coordinates of the meshpoints; CORG(1,2*NP+1) stores the grid at time level 0, i.e. the fixed grid; new grid coordinates are put into CORG(1,NP+1) and the grid velocity is computed accordingly
[out]XYZDOTCartesian components of the nodal grid velocities
[in]IBNDPTRBoundary informations: IBNDFAC(1,j) gives the element the j-th face belongs to; IBNDFAC(2,j) gives the local vertex number of element IBNDFAC(1,j) opposite the boundary face; IBNDFAC(3,j) gives the color of the boundary face
[in]NDIMdimension of the space
[in]NOFVERTnof boundary faces
[in]NELEMnof boundary faces
[in]NPOINnof interior nodes in the mesh
[in]NGHOSTnof ghost nodes in the mesh
[in]NPNODnof periodic nodes in the mesh
[in]NFACEnof faces in the mesh
[in]NBFACnof boundary faces
[in]NBINTnof inter-processor faces
[in]NHOLEnof holes in the mesh (should be 0 in 3D)
[in]TIMEphysical time to be reached, i.e. time .

Upon entry:

    LCORG       |---- n   ----| |---- XXX ----| |---- 0 ----|
    LMEDIAN     |---- n   ----| |---- XXX ----| |---- n ----|
    LVOL        |---- n   ----| |---- XXX ----| |----n-1----|
    LFACNOR     |---- n   ----|

    XXX means that the entry is irrelevant

Upon return:

    LCORG       |---- n+a ----| |---- n+1 ----| |---- 0 ----|
    LMEDIAN     |---- n   ----| |---- n+1 ----| |---- n ----|
    LVOL        |---- n+a ----| |---- n+1 ----| |----n-1----|
    LFACNOR     |---- n+a ----|

where n stands for the time level and a is the ALFALE parameter which is set through -ale_grid_weight
