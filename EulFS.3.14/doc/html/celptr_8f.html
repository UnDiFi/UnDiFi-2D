Purpose

retrieve data for cell IELEM from global arrays

Parameters
[in]IELEMis the current cell in local (per processor) numbering
[in]NELEMnof cells on the current PE
[in]ICELNODCell to node pointer: ICELNOD(i,j) gives the global node number of the i-th vertex of the j-th element
[in]ICELFACCell to face pointer: ICELFAC(i,j} gives the global face number of the face opposite the i-th vertex of the j-th element
[in]VOLarea/volume of the simplicial elements (triangles,tetrahedra)
[in]ZROEstores the dependent NOFVAR variables within all meshpoints; ZROE(1:NOFVAR,1,*) stores the values at time level n+1,k ; ZROE(1:NOFVAR,1,*) those at tme level n and ZROE(1:NOFVAR,3,*) those at time level n-1; could actually feed any other vector using the same layout, such as, e.g. the nodal coordinates; however, there must be room enough in the unsteady case
[in]FACNORCartesian components of the normals to a face, multiplied by the face area
[in]XYZDOTCartesian components of the nodal grid velocities
[in]NDIMdimension of the space
[in]NOFVERTnof boundary faces
[in]NOFVARnof dofs within each meshpoint
[in]LDAsecond leading dimension of ZROE, should equal NPOIN+NGHOST+NPNOD
[out]ICNreturns the vertices of the current simplex in 0-based indexing; to be used to insert values into PETSc vecs and/or mats
[out]VCZreturns the NOFVAR dofs of the NOFVERT vertices of cell IELEM; VCZ(1:NOFVAR,*,1) stores the values at time level n+1,k ; VCZ(1:NOFVAR,*,2) those at tme level n and VCZ(1:NOFVAR,*,3) those at time level n-1
[out]VCNreturns the NDIM Cartesian components of the NOFVERT faces of cell IELEM
[out]VCBreturns the NDIM Cartesian components of the nodal grid velocities at time n+1/2 of the NOFVERT vertices of cell IELEM; only if LALE is .TRUE.
[out]VOLUMEreturns the volume of simplex IELEM
